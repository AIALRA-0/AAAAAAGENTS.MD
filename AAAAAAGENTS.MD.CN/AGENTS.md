# AGENTS

## 说明
- 本文件用于索引项目约束，规定 Agent 在本项目中的执行流程、可编辑范围、校验规则与收尾标准

## 约束文件索引
- `./AGENTS.md`: 这是整个项目的 AI Agent 总控制文件，用于说明整体工作流程、任务执行规则以及各文档之间的关系，如果一个 AI 或开发者进入项目，第一步应该先看这个文件
- `./BACKGROUND.md`: 这个文件记录项目背景、用户与商业判断、可行性与技术环境，用于在执行前明确问题边界与目标颗粒度
- `./RESOURCE.md`: 这个文件记录当前可确认的本地资源、外部资源与凭据位置，便于执行时快速定位上下文输入
- `./MILESTONE.md`: 这个文件用于规划和跟踪项目的阶段性目标与进度状态，记录当前完成情况以及下一步计划
- `./CHANGE.md`: 这个文件用于以 REACT 结构记录每一次重要更新和修改原因、内容及影响，作为项目的结构化变更日志
- `./TREE.md`: 这个文件记录 项目完整的目录结构以及各文件的用途或元信息，用于帮助快速理解项目整体结构
- `./agents_standards/*_STANDARD.md`: 这些文件定义项目中特定类型文件的统一编写规范和结构要求，以确保风格一致且易于维护

## 编辑权限
- 只读策略通过收尾校验脚本软约束执行，若越界修改，Agent 必须恢复到无报错状态才可结束任务
- Agent 可编辑:
  - `./MILESTONE.md`
  - `./CHANGE.md`
  - `./TREE.md`
- Agent 只读:
  - `./AGENTS.md`
  - `./BACKGROUND.md`
  - `./RESOURCE.md`
  - `./agents_standards/*_STANDARD.md`

## Agent 标准工作流
1. 首先读取 `./AGENTS.md`、`./BACKGROUND.md`、`./RESOURCE.md` 以及相关的 `./agents_standards/*_STANDARD.md` 文件，以确认项目任务边界、上下文与规范要求
2. 执行前置就绪检查，`BACKGROUND`、`RESOURCE`、初始 `MILESTONE` 三者若存在未填写或占位内容，必须暂停实现并提示用户先补齐
3. 根据任务要求进行实现和执行，并且只允许修改被授权的文件或具体代码实现文件
4. 以第三方评审视角，审查本次工作是否严格满足 `MILESTONE` 当前节点的全部要求；如果全部严格满足，更改该节点状态为 `done`，如果没有严格满足，则返回步骤 3 继续迭代，或暂停并要求用户提供补充资源
5. 在完成所有任务后更新 `./MILESTONE.md`，并确保每一次任务只执行并更新一个里程碑节点
6. 在 `./CHANGE.md` 中记录本次修改，并按照REACT结构添加结构化变更记录
7. 执行命令 `python agents_tools/tree.py sync`，更新项目文件树，并补充填写所有缺失的项目文件结构信息
8. 执行命令 `python agents_tools/verify_rules.py finalize --fix-safe` 对项目进行最终规则校验，并自动进行安全范围内修复
9. 如果校验仍然报错，则继续修复问题并重复执行 `finalize`，直到命令返回状态码 `0` 为止
10. 输出本次执行过程中的流程优化建议，单行分条列出，每条格式为：`问题名称：问题现象、影响范围、根因判断、改进动作、预期收益、实施优先级（`P0`/`P1`/`P2`）`，必须可执行、可验证，禁止空泛表述
11. 通过执行 Git Push 将本地项目同步到远程仓库（当前流程中禁用该步骤）

## 文件树生成流程
1. 自动扫描项目根目录下的所有文件和目录，包括隐藏文件和临时文件
2. 将扫描到的新文件自动添加到 `yaml` 记录中，并为其创建条目，其中 `note` 字段默认留空
3. 返回给 Agent 所有 `note` 为空的 `yaml` 条目文件路径，并要求 Agent 对这些文件补充手动注释说明， `note` 需要填补的信息是文件在项目里面作用的核心描述，禁止 `note` 单纯填写文件元属性相关信息
4. 要求 Agent 以统一的中间格式返回键值对，并根据这些键值对将对应的 `note` 内容写入相应的文件 `yaml` 记录
5. 自动扫描并使用脚本更新所有文件的元信息字段，包括 `path`、`name`、`parent`、`type`、`status`、`last_modified` 和 `editable` 等参数
6. 根据最新数据自动生成当前项目的text 文件树视图，该视图无需展示已删除的文件，只展示当前最新的文件结构

## 校验流程
1. `./AGENTS.md` 没有遭受任何形式的修改
2. `./BACKGROUND.md` 没有遭受任何形式的修改
3. `./RESOURCE.md` 没有遭受任何形式的修改
4. `./agents_standards/*_STANDARD.md` 没有遭受任何形式的修改
5. `./agents_standards` 目录中不得新增任何在约束文件索引中不存在的文件
6. `./agents_tools` 目录中，除了`./agents_tools/__pycache__`内，不得新增任何此前不存在的文件，且没有文件遭受任何形式的修改
7. `./agents_web` 目录中，除了`./agents_web/__pycache__`内，不得新增任何此前不存在的文件，且没有文件遭受任何形式的修改
8. `./agents_artifacts` 目录中，只有`./agents_artifacts/logs`，`./agents_artifacts/notes`，`./agents_artifacts/outputs`三个子目录，不得出现任何其他文件夹或残留文件
9. `./BACKGROUND.md` 与 `./RESOURCE.md` 必须满足模板结构完整性，且所有必填字段必须非空、非占位词、非纯符号
10. `./MILESTONE.md` 中不得新增或删除任何里程碑节点，并且必须且只能有一个里程碑节点发生状态变化，变更记录的字段要严格符合要求，不能多，不能少，不能空缺内容，且所有节点的完成顺序必须符合既定逻辑顺序，任何后续节点都不得在其前置节点完成之前被标记为完成
11. `./CHANGE.md` 中的历史变更记录不得被修改或删除，并且必须新增且仅新增一条最新的变更记录，变更记录的字段要严格符合要求，不能多，不能少，不能空缺内容
12. `./TREE.md` 中的文件树条目不得被删除，对于已删除的文件必须只通过 `status` 字段记录状态变化，而不得直接移除条目
13. `./TREE.md` 中的文件树text视图与 `yaml` 中`status`有效的条目必须一一对应且完全同步，不得存在缺漏或不一致
14. `./TREE.md` 中 `yaml` 的每一个文件条目必须包含 `note` 字段，并且该字段不得为空
15. 更新上述所有相关文件的 `last_updated` 参数为最新时间戳
16. 检查并确保不存在文件内容被修改但未更新 `last_updated` 时间戳的错误文件残留
17. 检查所有文件头部的元数据格式必须正确完整，不得存在缺失字段或多余字段

## 失败处理
- 当任意校验步骤失败时，Agent 不得直接结束任务，必须继续执行修复流程，直到校验脚本返回状态码 `0` 为止
- 对于可以确定修复的问题，可以使用 `--fix-safe` 参数执行自动修复
- 对于无法自动确定修复方式的问题，必须由 Agent 进行显式修复，并在修复后重新执行校验

## 本地可视化 Web
- 可视化 Web 服务仅允许由用户手动启动，用于辅助用户观察和调试 Agent 的工作流程
- 在启动服务之前，需要先安装依赖 `pip install -r requirements.txt`
- Linux 一键启动本地服务的命令为 `./start_web.sh`
- Windows 一键启动本地服务的命令为 `./start_web.bat`
- Python 一键启动本地服务的命令为 `python ./start_web.py`
