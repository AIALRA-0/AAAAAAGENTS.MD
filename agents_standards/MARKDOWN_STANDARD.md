# MARKDOWN STANDARD

## 基本要求
- 所有文档必须使用 `UTF-8` 编码；
- 全文禁止使用中文句号 `。` 和英文句号 `.`，所有句末统一使用分号 `；`；
- 文档内容必须简洁明确，禁止使用空泛、模糊、口语化表达；
- 文档内容必须保持可读、可维护、可解析；
- 同一文档中的语气、术语、字段风格必须保持一致；

## 标题要求
- 所有文档必须使用标准 Markdown 标题语法；
- 每个文档只允许存在一个一级标题 `#`；
- 二级标题名称必须简短、明确、稳定；
- 标题层级不得跳级；

## 列表要求
- 所有正文内容统一使用 `-` 列表表达；
- 所有缩进统一使用 `-` 列表缩进；
- 禁止混用 `*`、`+`、有序列表或其他缩进形式；
- 同级列表的缩进宽度必须一致；
- 下级列表必须只比上级列表多一级缩进；
- 长内容必须拆分为多个短列表项，禁止堆叠成长段落；

## 标记要求
- 所有代码块都必须标注语言类型，例如 `markdown`、`yaml`、`json`、`bash`、`python`、`text`；
- 所有代码块、文件路径、变量名、字段名、命令、标识符都必须使用反引号包裹；
- 所有示例中的配置键、状态值、参数名、目录名都必须使用反引号包裹；
- 所有路径表达必须保持统一格式；
- 所有变量命名必须稳定一致，禁止同一概念反复更换名称；
- 所有字段名必须固定，禁止随意改名；

## 公式要求
- 所有 LaTeX 行内公式必须使用 `$...$`；
- 所有 LaTeX 块级公式必须使用 `$$...$$`；
- 禁止使用非标准公式包裹方式；
- 公式中的变量命名必须与正文保持一致；

## 内容要求
- 需要机器解析的内容必须使用可稳定解析的格式；
- 结构化内容必须保证键名稳定、层级清晰、格式一致；
- 必填字段不得留空；
- 非必填字段是否允许为空，必须事先定义清楚；
- 所有示例都必须尽量最小化，只保留必要内容；
- 错误示例和正确示例必须清晰区分；
- 禁止在示例中加入与当前规则无关的复杂内容；

## 一致性要求
- 中英文混排时，术语表达必须前后一致；
- 同一概念在同一文档中只能使用一种命名；
- 规则表达必须优先使用“必须”“禁止”“不得”“仅允许”等明确措辞；
- 禁止使用“差不多”“大概”“以后再补”这类不确定表达；
- 禁止留下无意义占位内容，例如“待补充”“暂无”“以后再说”；
- 自动化依赖的内容不得仅依赖视觉排版判断；

## 禁止事项
- 禁止使用句号；
- 禁止使用三级标题及以下标题；
- 禁止使用数字编号标题；
- 禁止不加反引号直接书写代码块、路径、变量、字段、命令或标识符；
- 禁止混用多套不一致的格式风格；
- 禁止使用会影响解析稳定性的自由格式写法；
- 禁止使用与内容无关的装饰性排版；
- 禁止留下语义不完整、结构不完整或规则不完整的内容；
